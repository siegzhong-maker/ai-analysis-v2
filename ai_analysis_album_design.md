# 相册侧 AI 分析入口及体验设计规划

## 1. 设计目标与核心理念
- **核心理念**：将“视频”与“AI分析”深度绑定，传达“视频经由AI处理后能产生高价值的数据资产（如集锦、比分、球员数据等）”的心智。
- **设计目标**：通过平滑的新手引导（胶囊态 -> 运营位 -> 专属Tab），降低用户认知门槛，提升AI分析功能的使用率和转化率。

## 2. 用户体验路径 (User Flow)

### 阶段一：首次曝光（胶囊态引导）
- **触发条件**：用户首次进入“相册”页面，且存在支持AI分析的视频，或系统希望引导用户体验。
- **UI 表现**：
  - 在相册子Tab栏（全部、视频、图片、成片）的右侧或下方，悬浮/嵌入一个醒目的**“胶囊态”按钮**（例如：“✨ 探索AI分析”或带有动态扫光效果的胶囊）。
  - 或者在列表流中插入一个特殊的卡片作为胶囊态引导。
- **交互**：点击该胶囊态。

### 阶段二：价值传递（运营位介绍）
- **触发条件**：点击首次出现的“胶囊态”。
- **UI 表现**：
  - 弹出半屏面板（Bottom Sheet）或全屏沉浸式介绍页。
  - **内容展示**：动图或短视频演示AI分析的核心能力（例如：自动捕捉进球瞬间、生成A队/B队比分面板、球员跑动热力图等），明确告知用户“你能获得什么”。
- **交互**：点击底部的“立即开启”或“查看我的AI资产”按钮。

### 阶段三：常态化专属阵地（新增专属栏目）
- **触发条件**：完成阶段二的引导后。
- **UI 表现**：
  - 胶囊态消失。
  - 相册子Tab栏目永久增加一项：**“AI分析”**（排列在“成片”之后或“视频”之后）。
  - **列表视觉（参考当前UI）**：
    - 在“AI分析”Tab下，聚合所有已完成AI分析或正在分析的视频。
    - **视频封面强化**：如视觉稿所示，已完成分析的视频封面右上角带有醒目的橙色 **[AI]** 角标。
    - **数据外显**：封面直接透出核心分析结果（例如：A队 2 : B队 1），强化“数据资产”的概念。

## 3. 核心场景：视频与AI资产的绑定
- **资产生成**：在普通“视频”Tab中，用户可以选中某段足球比赛视频，发起“AI分析”任务。
- **资产查看**：
  - 点击带有 **[AI]** 角标的视频，进入专属的**AI播放器/详情页**。
  - 该页面不仅包含视频播放，还深度融合了数据资产（时间轴上的高光打点、两队比分板、球员数据统计面板等）。

## 4. 后续开发与落地拆解建议
1. **UI组件开发**：
   - 胶囊态引导组件 (Capsule Guide)
   - 运营位介绍弹窗 (Marketing Bottom Sheet)
   - 相册顶部 Tab 动态扩展逻辑 (支持动态增加“AI分析” Tab)
   - 视频封面卡片组件升级 (支持 [AI] 角标及比分数据 Overlay 渲染)
2. **状态管理**：
   - 用户首次进入状态记录 (Local Storage / 接口下发)
   - AI分析任务状态轮询与列表状态更新 (分析中、分析完成、分析失败)
3. **路由与页面**：
   - 新增 `AIAnalysisTab` 视图容器。
   - 跳转至 `AIDetailPlayer` (AI专属播放与数据详情页) 的路由逻辑。
