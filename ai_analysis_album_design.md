# 相册侧赛事解读入口及体验设计规划

## 1. 设计目标与核心理念
- **核心理念**：将“视频”与“赛事解读”深度绑定，传达“视频经由云端处理后能产生高价值的数据资产（如集锦、比分、球员数据等）”的心智。
- **设计目标**：通过平滑的新手引导（胶囊态 -> 运营位 -> 专属Tab），降低用户认知门槛，提升赛事解读功能的使用率和转化率；同时针对云端分析的链路（Falcon -> Cloud）提供清晰的状态反馈与异常处理。

## 2. 用户体验路径 (User Flow)

### 阶段一：首次曝光（胶囊态引导）
- **触发条件**：用户首次进入“相册”页面，且存在支持解读的比赛视频，或系统希望引导用户体验。
- **UI 表现**：
  - 在相册子Tab栏（全部、视频、图片、成片）的右侧或下方，悬浮/嵌入一个醒目的**“胶囊态”按钮**。
  - **文案设计**：去工程化，贴合实际业务场景。例如：**“✨ 开启赛事解读”**、**“生成专属战报”** 或 **“解锁高光时刻”**。
- **交互**：点击该胶囊态。

### 阶段二：价值传递（运营位介绍）
- **触发条件**：点击首次出现的“胶囊态”。
- **UI 表现**：
  - 弹出半屏面板（Bottom Sheet）或全屏沉浸式介绍页。
  - **内容展示**：动图或短视频演示赛事解读的核心能力（例如：自动捕捉进球瞬间、生成A队/B队比分面板、球员跑动热力图等），明确告知用户“你能获得什么”。
- **交互**：点击底部的 **“立即开启”** 按钮。

### 阶段三：常态化专属阵地与项目整合
- **触发条件**：完成阶段二的引导后。
- **UI 表现**：
  - 胶囊态消失。
  - 相册子Tab栏目永久增加一项：**“赛事解读”**（或“赛事战报”，替代原工程化的“AI分析”）。
  - **状态聚合与“我的项目”整合**：
    - 现有的顶部**“我的项目”**沿用，并将“赛事解读”的上云及处理任务（Falcon上传中、云端分析中）无缝整合至其中。用户可以通过顶部的“我的项目”入口，集中查看从 Falcon 设备同步到云端（Cloud）的整体进度。
  - **列表视觉**：
    - 在“赛事解读”Tab下，聚合所有相关的视频。
    - 视频封面强化：已完成的视频封面右上角带有醒目的专属角标（如 **[战报]** 或 **[解读]**），封面直接透出核心比赛数据（如 A队 2 : B队 1）。

## 3. 云端分析异常处理与状态呈现 (Falcon -> Cloud)
由于赛事解读依赖将视频从 Falcon 传至云端分析，需完整覆盖设备同步、云端计算中的各种异常情况：

### 3.1 异常场景拆解
1. **上传阶段异常 (Falcon -> Cloud)**：网络中断、设备断开连接、云端存储空间不足等导致视频未能完整上云。
2. **分析阶段异常 (Cloud 端)**：视频内容不符合要求（如非足球比赛、画质极度模糊、镜头严重遮挡）、云端算力排队超时、算法解析报错。

### 3.2 异常样式与交互交付
- **列表卡片样式 (UI)**：
  - **异常视觉遮罩**：在“赛事解读”列表或“全部”列表中，分析失败的视频卡片覆盖一层半透明暗色/红色警示遮罩。
  - **错误提示**：中心或醒目位置展示“叹号”错误图标及简短提示词（如“上传中断”、“解读失败”）。
- **错误详情与重试机制 (Interaction)**：
  - **外显原因**：点击卡片或在卡片底部显示具体原因（例如：“设备断开，请检查 Falcon 连接”、“视频过于模糊，无法提取比赛数据”）。
  - **一键重试**：卡片上提供明确的 **“重试”** 按钮（支持“重新上传”或“重新分析”）。
  - **降级处理**：针对明确不支持分析的视频（如被算法判定非比赛视频），提供“查看支持规范”说明，并允许用户关闭该视频的解读状态，作为普通视频保留。
- **“我的项目”联动**：
  - 在顶部“我的项目”任务列表中，将失败的任务**置顶飘红**展示，方便用户集中处理所有设备的异常上传和分析任务。

## 4. 核心场景：视频与赛事资产的绑定
- **资产查看**：点击带有解读标识的视频，进入专属的**赛事播放详情页**。页面不仅包含视频播放，还深度融合了时间轴上的高光打点、两队比分板、球员数据统计面板等，真正将普通视频升级为“数据资产”。
